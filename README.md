# 🍱 企業便當訂餐系統 (Meal Ordering System)

這是一個基於 **GitHub Pages** 前端與 **Supabase** 後端開發的輕量化企業購餐登記系統。系統專為簡化辦公室每日訂餐流程而設計，具備自動對帳與台灣時區校正功能。

## 🚀 系統核心特點

* **模組化設計**：前端介面、餐點設定、資料庫邏輯完全分離，易於維護。
* **台灣時區校正**：強制使用 `Asia/Taipei` (UTC+8) 判定訂餐日期，解決跨日顯示錯誤。
* **動態餐點維護**：透過外部 `js/meals.js` 檔案即可輕鬆調整菜色與價格。
* **自動化對帳**：提供個人月度統計與管理員全員對帳報表。
* **響應式介面**：適配手機與電腦操作，符合員工日常使用習慣。

## 🛠️ 維護說明

### 如何調整餐點與價格？

請直接編輯 `js/meals.js` 檔案，格式如下：

```javascript
const MEAL_CONFIG = [
    { name: "排骨飯", price: 120 },
    { name: "雞排飯", price: 120 },
    { name: "雞腿飯", price: 130 },
    { name: "素食便當", price: 70 }
];

```

修改後存檔推送到 GitHub，系統將自動同步更新下拉選單、計算逻辑與後台報表。

## 📊 資料庫配置 (Supabase)

系統使用 Supabase 儲存數據，請確保資料庫中存在以下資料表：

### `meal_orders` 資料表

| 欄位名稱 | 類型 | 說明 |
| --- | --- | --- |
| `id` | int8 | 自動遞增 ID (Primary Key) |
| `user_email` | text | 員工帳號 (Email) |
| `user_name` | text | 員工姓名 |
| `meal_option` | text | 點餐品項名稱 |
| `created_at` | timestamptz | 存檔時間 (預設為 UTC) |
